🚀 How to Launch Your Site

GitHub (free) → Vercel (free) · ~15 minutes total

1
Get a Fresh Anthropic API Key

Go to console.anthropic.com → API Keys → Create Key. Copy it — you’ll add it in Step 3.

Your key never touches the HTML file. It lives only in Vercel’s secure environment variables, invisible to anyone viewing your source code. This fully resolves the GitHub warning.
2
Set Up Your GitHub Repository

Go to github.com → New Repository → name it road-on → Public → Create.

Upload two files in this exact structure:

road-on/ ├── index.html └── api/ └── assess.js

Option A — No terminal: Click Add file → Upload files to add index.html. Then click Add file → Create new file, type api/assess.js as the filename (GitHub creates the folder automatically), and paste in the contents of your assess.js file.

Option B — Terminal:

mkdir road-on && cd road-on mkdir api # copy index.html and api/assess.js into place, then: git init git add . git commit -m "Road On secure launch" git branch -M main git remote add origin https://github.com/YOUR_USERNAME/road-on.git git push -u origin main
3
Deploy on Vercel & Add Your Secret Key

Go to vercel.com → Sign up with GitHub → Add New → Project → select road-on → Import.

Before clicking Deploy, scroll to Environment Variables and add:

Name: ROAD_ON_KEY Value: sk-ant-... (your Anthropic key)

Then click Deploy. ✓ Live in ~30 seconds at road-on.vercel.app

Already deployed? Go to your project → Settings → Environment Variables → add ROAD_ON_KEY → then Deployments → Redeploy.
4
Waitlist Form — Already Connected ✓

Your waitlist posts directly to your Google Form — free, no limits, every response lands in your Google Sheet automatically.

To view submissions: open your Google Form → Responses tab → click the Sheets icon.

5
Optional: Custom Domain

Buy a domain at namecheap.com — e.g. roadon.ng or roadon.africa (~$10–15/yr). In Vercel → Settings → Domains → add it and follow the DNS instructions.

✓
Pre-Launch Checklist
  • Created a fresh Anthropic API key (revoke any previously exposed keys)
  • Added ROAD_ON_KEY as a Vercel Environment Variable
  • Uploaded both index.html and api/assess.js to GitHub
  • Tested the 4-step AI assessment all the way through on the live URL
  • Tested the waitlist form — checked your Google Sheet for the entry
  • Checked the site on your phone
  • Shared the live URL in diaspora WhatsApp groups and LinkedIn
  • Included the live URL in your TEF application as MVP evidence
